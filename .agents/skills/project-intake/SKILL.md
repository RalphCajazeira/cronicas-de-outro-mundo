# Skill: Project Intake

## Objetivo

Classificar o estado do projeto antes de propor código.

## Verificar

1. O repositório está vazio, iniciado ou legado?
2. Há `AGENTS.md`, `docs/ai/`, README e scripts?
3. Qual é a stack real?
4. Existe backend, frontend, mobile, banco ou infra?
5. Qual gerenciador de pacotes existe? Não trocar sem autorização.
6. Quais scripts reais podem validar o projeto?
7. Há sinais de produção, dados reais, migrations ou integrações sensíveis?

## Entrega

- classificação do projeto;
- stack detectada;
- estrutura atual;
- lacunas de documentação;
- primeira task segura recomendada;
- perguntas mínimas para destravar a Fase 0.

Não instale dependências e não altere arquivos durante a triagem, salvo se o prompt autorizar.
