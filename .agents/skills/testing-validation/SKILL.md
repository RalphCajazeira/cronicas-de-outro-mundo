# Skill: Testing and Validation

## Processo

1. Descobrir scripts reais no `package.json` ou equivalente.
2. Não inventar comandos inexistentes.
3. Escolher validações compatíveis com o escopo.
4. Rodar testes focados antes de suíte ampla quando possível.
5. Registrar validações não executadas e o motivo.

## Preferências Node/TypeScript

- Backend: Vitest + Supertest.
- Frontend: Vitest + Testing Library + jsdom.
- E2E: Playwright apenas para fluxos críticos.

## Entrega

- comandos executados;
- resultado;
- falhas e causa provável;
- se falha é do escopo atual ou preexistente;
- próximo passo recomendado.
