# Skill: Architecture Refactor

## Princípios

- Refatorar em etapas pequenas.
- Preservar comportamento existente.
- Reutilizar código antes de criar estruturas novas.
- Evitar reescrita total sem autorização explícita.
- Não misturar refactor amplo com feature nova.

## Antes de alterar

1. Mapear estrutura atual.
2. Identificar padrões reais já existentes.
3. Listar responsabilidades misturadas.
4. Propor plano incremental.
5. Definir arquivos que serão tocados na etapa atual.
6. Definir validações que garantem que o comportamento não mudou.

## Entrega

- objetivo da etapa;
- arquivos movidos/alterados;
- compatibilidade preservada;
- riscos;
- próxima etapa sugerida.
